# 公司資料傳輸工具 — 設計規格

日期:2026-07-19
作者:hpisokplay

## 背景與目標

公司禁止一切對外通訊軟體(LINE / IG / TG 等),只能用信箱,但 Email 傳大檔又慢又常失敗。經確認公司**未封鎖 GitHub**,且公司電腦可連 `api.github.com`。目標是做一個純網頁 / PWA 工具,把 Word / PPT / PDF / Excel / 圖片(通常 < 25MB)在**公司電腦 ↔ 手機**之間快速中轉,主要方向為電腦 → 手機。

## 關鍵約束

- 公司電腦:可開網頁、可連 `api.github.com`、可連 GitHub Pages。
- 手機:走行動網路 / 家用 WiFi,網路不受限。
- 檔案通常 < 25MB。
- 不得依賴 GitHub Desktop 或任何額外軟體,全部在瀏覽器完成。
- 檔案可能是機密(客戶文件),需加密。

## 整體架構

- **App 程式碼**:放公開的 GitHub Pages 儲存庫,做成可安裝的 PWA。
- **檔案資料**:放另一個**私人**儲存庫,透過 GitHub REST API + 個人存取金鑰(PAT)存取。
- **金鑰(PAT)**:每台裝置手動輸入一次,存在該裝置瀏覽器 `localStorage`,絕不寫進程式碼、絕不上傳。
- **完全在瀏覽器完成**上傳 / 下載 / 加解密,不需 git 客戶端。

## 安全設計(三層)

1. **PAT**:僅存本機,程式碼公開也看不到。私人資料庫只有持金鑰者能存取。
2. **私人儲存庫**:外人搜不到、看不到。
3. **端對端加密 AES-256-GCM**:
   - 內容在裝置端加密,GitHub 只存亂碼;連 GitHub 官方也解不開。
   - **檔名也加密**(存在加密的索引中),客戶名稱之類不外洩。
   - 金鑰由使用者自訂密碼經 PBKDF2 衍生;電腦與手機設同一組密碼。
   - 忘記密碼 → 舊檔無法解,但屬中轉用途,重設密碼重傳即可。

## 儲存結構(私人資料庫內)

```
/index.enc          加密後的 JSON 清單:[{id, name, category, size, time, sha, path}]
/data/{id}.bin      每個檔案的加密位元組
```

- 加密區塊格式:`[salt(16) | iv(12) | ciphertext]`。
- 讀檔用 Git Blobs API(`GET /git/blobs/{sha}`,支援到 100MB),避開 Contents API 讀取 >1MB 的限制。
- 寫檔用 Contents API(`PUT /contents/{path}`),回傳 sha 記入索引。
- 更新索引前先取最新 sha,遇 409 衝突重試一次(單人雙裝置,衝突機率低)。

## 分類(自動,依副檔名)

底部 6 個頁籤,上傳時自動歸位:

| 頁籤 | 內容 |
|------|------|
| 🕐 最近 | 全部檔案,依時間排序(含剛傳好的) |
| 🖼️ 圖片 | jpg / jpeg / png / gif / webp / heic |
| 📘 Word | doc / docx |
| 📙 PPT | ppt / pptx |
| 📕 PDF | pdf |
| 📗 Excel | xls / xlsx / csv |

其他副檔名歸「最近」。

## 操作流程

- **上傳**(電腦為主):選檔 / 拖曳(可多選)→ 逐檔加密 → 上傳 → 顯示進度 → 更新索引。
- **下載**(手機為主):點下載 → 抓 blob → 解密 → 觸發下載存入「檔案」App;圖片可用分享選單存相簿。
- **刪除**:每檔有刪除鈕;另可開「下載後自動刪除」保持乾淨。
- **設定**(齒輪):填 GitHub 帳號、私人庫名稱、PAT、加密密碼。設定存本機。

## 視覺設計(重點:質感)

- 風格:霧面玻璃 + 柔和層次陰影,大量留白。
- 主色:靛藍→紫漸層(#6366f1 → #8b5cf6);中性以 slate 灰階。
- 圓角 16–20px,柔和多層陰影;過場 200–300ms 平滑動畫。
- 淺色 / 深色自動切換(跟隨系統)。
- 檔案類型色碼:Word 藍、PPT 橘、PDF 紅、Excel 綠、圖片 紫粉。
- 微互動:hover 上浮、按壓縮放、上傳進度環、成功打勾動畫、Toast 提示。
- 底部頁籤:滑動式高亮指示。

## 交付物

- `index.html`(單檔完整 app,含 CSS / JS)
- `manifest.webmanifest`(PWA 設定)
- `sw.js`(Service Worker,離線外殼 + 可安裝)
- `icons/`(app 圖示)
- `README.md`(建庫、設定 PAT 權限、部署步驟)

## 非目標(YAGNI)

- 不做多使用者 / 帳號系統。
- 不做檔案預覽 / 線上編輯。
- 不做 >100MB 大檔(超出即提示)。
- 不做 5 個真實儲存庫(改用 1 私人庫 + 分類)。
